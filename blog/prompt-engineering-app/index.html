Todo App बनवा Claude Code ने — AI ला नक्की काय सांगायचं?

Claude Code सारख्या AI tools ने app बनवणं आता शक्य आहे — पण तुम्ही काय सांगता त्यावर सगळं अवलंबून आहे. चुकीचं prompt दिलं तर AI random choices करतो, आणि तुम्हाला नको ते मिळतं.

या tutorial मध्ये आपण एक Todo app बनवायला शिकणार — पण focus app वर नाही, तर prompt कसं लिहायचं यावर आहे. Same technique कोणत्याही app साठी वापरता येईल.

Part 1: Prompt कसं improve होतं — step by step

1. Bad prompt — "मला app बनवून दे"

Bad Prompt

"Make me a todo app"

हे prompt वाचून AI ला काहीच clarity नाही:

  • Platform कोणता? Web app? Android? iOS? Desktop? AI randomly निवडेल — कदाचित web app बनवेल जेव्हा तुम्हाला mobile हवं होतं
  • Tech stack? React Native? Flutter? Kotlin? AI स्वतःच ठरवेल — आणि तुम्हाला ते run करता येणार नाही कारण तुमच्याकडे ते setup नाही
  • Features काय? फक्त add/delete? Categories हव्या? Due dates? Reminders? AI guess करेल
  • Design कसं? Light theme? Dark? Colors कोणते? AI चा default output बहुतेक boring असतो

Result: तुम्हाला एक generic, boring app मिळेल जो तुम्हाला नको त्या tech stack मध्ये असेल. मग तुम्ही 10 follow-up prompts लिहून fix कराल — वेळ वाया.

2. Better prompt — details add करा

Better Prompt

"Make me an Android todo app using React Native. It should have: add tasks, mark complete, delete tasks. Use a dark theme."

हे बरंच चांगलं आहे कारण:

  • Platform clear आहे (Android)
  • Tech stack specified आहे (React Native)
  • Basic features listed आहेत
  • Design direction आहे (dark theme)

पण अजूनही important details missing आहेत:

  • Expo वापरायचं की bare React Native? (Expo ने testing सोपं होतं)
  • Data कुठे store करायचा? Local? Cloud?
  • UI layout कसं? Floating button? Tab bar?
  • Colors exact कोणते? "Dark theme" म्हणजे 1000 variations आहेत
  • Categories, due dates यासारखे useful features missing

3. Great prompt — complete spec द्या

Great Prompt — Ready to Use

हा prompt Claude Code मध्ये directly paste करा:

"मला एक Todo app बनवायचा आहे Android साठी. Details:

  • Tech: React Native with Expo (so I can test easily)
  • Features: add task with title + optional due date, mark complete (strikethrough), delete with swipe, categories (काम, शिक्षण, घर, इतर), filter by category
  • Design: dark theme, orange accent color (#D4915E), Marathi UI labels (जोडा, पूर्ण, काढा), clean minimal design
  • Data: store locally with AsyncStorage (no backend needed)
  • Screen: single screen with floating add button, task list with checkboxes

Give me the complete project. Start with expo init, then build each component."

हे prompt का काम करतं?

  • Tech stack exact आहे — React Native with Expo, म्हणजे AI ला कोणतं project structure बनवायचं ते माहीत आहे
  • Features list specific आहे — "add task" नाही, तर "add task with title + optional due date" — AI ला exact form fields माहीत आहेत
  • Design tokens दिले — exact hex color (#D4915E), theme (dark), style (minimal) — AI ला guess करायची गरज नाही
  • Data strategy clear आहे — AsyncStorage, no backend — AI server code बनवणार नाही
  • Layout described आहे — floating button, checkboxes — AI ला screen कसं दिसेल ते माहीत आहे
  • Step-by-step instruction — "Start with expo init, then build each component" — AI एकदम सगळं dump करणार नाही

4. Iteration — app बनवणं ही conversation आहे

एक perfect prompt दिलं तरी first output 100% perfect नसतो. App building ही conversation आहे — prompt दे, output बघ, adjust कर, पुढे जा.

पहिल्या generation नंतर असे follow-up prompts द्या:

Follow-up Prompt 1: Feature add करा

"Add a 'सगळे पूर्ण' button at the top to mark all tasks complete at once"

Follow-up Prompt 2: Bug fix / polish

"The delete animation is jerky — make it smooth with 300ms transition"

Follow-up Prompt 3: नवीन screen add करा

"Add a simple splash screen with the app name 'माझी यादी' in white text on dark background, show for 2 seconds"

Important: प्रत्येक step नंतर app run करा आणि test करा. एकदम 10 features मागू नका — एक-एक करा, test करा, मग पुढे जा.

5. Key Takeaways

कोणताही app बनवताना या 5 गोष्टी prompt मध्ये असायलाच हव्या:

  1. Tech stack specify करा — language, framework, आणि tools exact सांगा
  2. Features explicitly list करा — प्रत्येक feature चे details द्या, vague ठेवू नका
  3. Design tokens द्या — colors (hex codes), theme, font style, spacing
  4. Layout describe करा — कोणत्या screen वर काय दिसेल ते सांगा
  5. Small steps मध्ये iterate करा — एकदम सगळं मागू नका, step by step बनवा

Part 2: Ready-to-use prompts — Copy-paste करा आणि बनवा

खालचे prompts Claude Code मध्ये directly paste करता येतील. प्रत्येक prompt complete spec आहे — AI ला guess करायची गरज नाही.

Expense Tracker App — दैनंदिन खर्च track करा

"Build me an expense tracker Android app. Details:

  • Tech: React Native with Expo
  • Features: add expense with amount + description + date, categories (खाणे, प्रवास, बिलं, शॉपिंग, इतर), monthly total at top, list of expenses sorted by date, delete expense with swipe, filter by category, simple bar chart showing category-wise spending
  • Design: dark theme, green accent (#4CAF50) for income, red (#E57373) for expenses, Marathi UI labels (जोडा, एकूण, महिना), clean cards for each expense
  • Data: AsyncStorage, no backend
  • Screens: home screen with expense list + monthly total, add expense modal with category picker, simple stats screen with bar chart

Start with expo init, then build the home screen first."

Recipe App — मराठी रेसिपी save करा

"Build me a recipe app for Android. Details:

  • Tech: React Native with Expo
  • Features: add recipe with title + ingredients list + steps + cooking time + photo (from gallery), search recipes by name or ingredient, categories (नाश्ता, जेवण, snacks, गोड पदार्थ, पेये), favorite recipes, share recipe as text
  • Design: light theme, warm orange accent (#FF8A65), Marathi labels (रेसिपी जोडा, शोधा, आवडते), recipe cards with photo thumbnail, clean typography
  • Data: AsyncStorage for recipes, expo-image-picker for photos, stored locally
  • Screens: home with recipe grid, recipe detail with full ingredients and steps, add/edit recipe form, search screen with filters

Start with expo init, build the recipe list screen first, then add form."

Habit Tracker — रोज check-in करा, streak वाढवा

"Build me a habit tracker app for Android. Details:

  • Tech: React Native with Expo
  • Features: add habits with name + icon emoji, daily check-in (tap to mark done), streak counter for each habit (consecutive days), weekly view showing 7 days with done/not-done indicators, reset streak if missed, simple stats (current streak, best streak, completion rate)
  • Design: dark theme, purple accent (#B39DDB), Marathi labels (सवय जोडा, आजचं, streak), habit cards with big tap area, satisfying checkmark animation on completion
  • Data: AsyncStorage, store daily completion log per habit
  • Screens: home with today's habits and streaks, add habit form, habit detail with weekly calendar view and stats

Start with expo init, build the daily check-in screen first."

Calculator with History — calculation history save करा

"Build me a calculator app for Android with history. Details:

  • Tech: React Native with Expo
  • Features: basic calculator (+, -, x, /, %), history of past calculations, tap on history item to reuse that number, clear history button, copy result to clipboard with long press
  • Design: dark theme, teal accent (#4DB6AC), large buttons with good tap targets, display at top showing current input and result, history slides up from bottom as a panel, Marathi labels for clear (पुसा) and history (इतिहास)
  • Data: AsyncStorage for history, last 50 calculations
  • Screens: single screen — calculator top half, history panel bottom half (collapsible)

Start with expo init, build the calculator logic first, then add history."

Notes App with Search — quick notes लिहा, शोधा

"Build me a notes app for Android. Details:

  • Tech: React Native with Expo
  • Features: create note with title + body text, edit existing notes, delete with swipe, search notes by title or content (real-time filtering as you type), pin important notes to top, sort by date modified, note preview in list (first 2 lines of body)
  • Design: dark theme, yellow accent (#FFD54F), Marathi labels (नवीन नोट, शोधा, pin करा), note cards with title bold and preview below, search bar at top, floating add button
  • Data: AsyncStorage, each note stored with id, title, body, timestamp, pinned boolean
  • Screens: home with notes list and search bar, note editor (full screen), no separate view screen — tap to edit directly

Start with expo init, build the notes list first, then add editor."

Part 3: सगळ्यात common चुका

App बनवताना हे mistakes करू नका:

1. Tech stack specify करू नका

"Make me an app" म्हणलात तर AI कदाचित Swift मध्ये iOS app बनवेल — आणि तुमच्याकडे Mac नसेल तर run च करता येणार नाही. नेहमी platform + framework + tools सांगा.

2. एकदम सगळं एकाच prompt मध्ये मागणं

"Todo app बनव with push notifications, cloud sync, user login, dark mode, animations, and 5 screens" — हे एका prompt मध्ये दिलं तर output messy येतो. Core features आधी बनवा, बाकी iterate करत add करा.

3. UI layout describe करू नका

तुम्ही "add a list of tasks" म्हणलात — पण list कुठे? Screen च्या top ला? Bottom ला? Floating button हवा की tab bar? Layout सांगा — कोणत्या screen वर काय, कुठे, कसं.

4. Data storage बद्दल काहीच न सांगणं

App चा data कुठे save होतो हे सांगितलं नाही तर AI कदाचित Firebase setup करेल — ज्याला account लागतो, configuration लागतं. Simple app साठी AsyncStorage (local) पुरेसं आहे — explicitly सांगा.

5. प्रत्येक step test करू नका

5 features एकदम add केल्या आणि मग app crash झालं — कोणत्या feature मुळे ते शोधणं कठीण. प्रत्येक step नंतर run करा, test करा, मग पुढचा step मागा.

लक्षात ठेवा: AI ला app बनवायला सांगणं म्हणजे एखाद्या builder ला घर बनवायला सांगणं. "घर बनव" म्हणलात तर काहीही मिळेल. पण "2BHK, east-facing, modular kitchen, vitrified tiles" म्हणलात तर तुम्हाला हवं ते मिळेल. Prompt मध्ये detail द्या — AI बरोबर बनवेल.
